# AI Security Glossary

Key terminology used in AI security, adversarial ML, and LLM safety.

## Overview

This glossary defines 35+ terms commonly used in AI security. Terms are organized alphabetically for quick reference.

## Terms

| Term | Definition |
|------|-----------|
| **Adversarial Example** | Input deliberately crafted to cause a model to make incorrect predictions |
| **Alignment** | Ensuring AI systems behave in accordance with human intent and values |
| **AI-SBOM** | AI Software Bill of Materials — inventory of all components in an AI system |
| **Backdoor** | Hidden trigger in a trained model that activates attacker-chosen behavior |
| **Chain-of-Thought (CoT) Attack** | Manipulating model reasoning through crafted intermediate steps |
| **Concept Drift** | Change in the relationship between inputs and outputs over time |
| **Confidential Computing** | Hardware-based isolation for data in use (TEEs, SGX, TDX) |
| **Data Drift** | Change in input data distribution compared to training data |
| **Data Poisoning** | Corrupting training data to manipulate model behavior |
| **Denial-of-Wallet** | Attack that causes excessive API costs to the victim |
| **Differential Privacy** | Mathematical framework for quantifying privacy loss in data analysis |
| **Embedding** | Dense vector representation of text, images, or other data |
| **Evasion Attack** | Modifying inputs at inference time to avoid correct classification |
| **Exfiltration** | Unauthorized extraction of data from an AI system |
| **Fine-tuning** | Adapting a pre-trained model to a specific task with additional training |
| **Guardrails** | Runtime safety boundaries that constrain LLM behavior |
| **Hallucination** | AI generating confident but factually incorrect information |
| **Indirect Prompt Injection** | Injection via external content (documents, web pages) retrieved by the model |
| **Jailbreak** | Bypassing safety alignment to make a model produce restricted outputs |
| **Kill Switch** | Emergency mechanism to immediately disable an AI system |
| **LoRA** | Low-Rank Adaptation — efficient fine-tuning technique |
| **Model Card** | Documentation of a model's intended use, performance, and limitations |
| **Model Extraction** | Stealing a model's functionality through query access |
| **Model Inversion** | Reconstructing training data from model outputs |
| **Prompt Injection** | Manipulating LLM behavior by inserting instructions in user input |
| **RAG** | Retrieval-Augmented Generation — combining retrieval with generation |
| **Red Teaming** | Adversarial testing to find vulnerabilities before attackers do |
| **RLHF** | Reinforcement Learning from Human Feedback — alignment technique |
| **SafeTensors** | Secure model serialization format that prevents code execution |
| **Shadow Model** | Replica model trained to mimic a target model's behavior |
| **System Prompt** | Hidden instructions that define LLM behavior and constraints |
| **Temperature** | Parameter controlling randomness in model output generation |
| **Token** | Smallest unit of text processed by an LLM (word piece or character) |
| **Transfer Attack** | Adversarial example crafted on one model that works on another |
| **Trojan** | Malicious functionality hidden in a model that activates on specific triggers |
| **Vector Database** | Database optimized for storing and querying embedding vectors |
| **Zero Trust** | Security model assuming no implicit trust for any entity or component |
